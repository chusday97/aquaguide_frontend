# AquaGuide CTA 与导航行为审计

## 目的

本文件是 AquaGuide 按钮、弹窗、路由和长页面定位的行为契约。任何可点击入口都必须产生至少一种可观察结果：

- 进入明确路由；
- 打开任务弹窗；
- 切换互斥视图；
- 保存或修改状态；
- 定位到明确板块并高亮；
- 显示成功或失败反馈。

如果点击后只有日志、滚到当前已经可见的内容、重复打开同一内容，或没有可见变化，该入口属于伪交互，应删除或降级为普通说明。

## 当前系统性问题

1. `DesktopSidebar` 使用 `setTimeout(80) + scrollIntoView` 定位 hash，无法保证目标已渲染，也没有使用桌面真实滚动容器。
2. `Aquarium`、`Encyclopedia`、`CareEncyclopedia` 分别实现自己的 `scrollIntoView`、`scrollTo` 和 hash 处理，目标偏移、失败反馈和返回行为不一致。
3. 图鉴与养护百科仍存在“查看内容”等无法预判结果的 CTA，且部分卡片同时支持整卡点击和内部重复按钮。
4. 详情弹窗关闭后主要依赖组件状态，未统一恢复来源筛选、页码、滚动位置和触发焦点。
5. 长页面目标没有统一 section 注册表；sticky 工具区可能遮住定位后的标题。

## 动作类型

| 类型 | 说明 | 必须反馈 |
| --- | --- | --- |
| `route` | 进入独立页面或任务模式 | URL 与页面标题变化 |
| `dialog` | 打开独立任务弹窗 | 弹窗标题、焦点进入弹窗 |
| `view` | 切换互斥 Tab、结果或诊断状态 | 选中态和内容变化 |
| `mutation` | 保存、记录、收藏、删除 | loading、成功或失败反馈 |
| `section` | 定位到长页面明确板块 | 正确滚动、高亮和焦点 |
| `external` | 打开外部资源 | 明确外链标识 |

## 长页面 Section 契约

只允许注册以下稳定目标：

```text
aquarium-overview
aquarium-actions
aquarium-tank
aquarium-records
aquarium-discovery

atlas-toolbar
atlas-results
atlas-grid
atlas-pagination-bottom

care-search
care-categories
care-results
care-diagnosis
care-actions
```

定位时必须使用当前页面真实滚动容器，并完成：

1. 读取 sticky 区域实际高度；
2. 预留 20px 顶部安全距离；
3. 定位后短暂高亮目标；
4. 将焦点移到目标标题；
5. 目标不存在时显示错误 Toast；
6. 不使用固定毫秒延迟等待 DOM。

## 返回上下文契约

打开详情或二级任务前保存：

```ts
interface NavigationContext {
  route: string;
  query: string;
  hash: string;
  scrollTop: number;
  page?: number;
  selectedTab?: string;
  sourceId?: string;
}
```

关闭后恢复：

- 来源页面；
- 搜索和筛选；
- 分页；
- 列表滚动位置；
- 触发按钮焦点。

不把临时 Toast、loading 或未确认弹窗状态写入 URL。

## 第一实施批次

1. 删除 Sidebar 的固定延迟滚动，接入统一 section 导航。
2. 统一桌面和手机滚动容器解析。
3. 先接入鱼缸常用操作、图鉴分页和养护分类三条路径。
4. 删除养护推荐卡和文章卡中的重复“查看内容”。
5. 为关闭详情恢复焦点和列表位置增加浏览器测试。

## 验收标准

- 任一主 CTA 点击后 300ms 内有视觉反馈；
- 不存在“查看内容”“立即处理”等模糊主 CTA；
- 不存在 handler 为空或仅输出日志的按钮；
- 长页面定位目标不被 sticky 区域遮挡；
- 关闭详情后恢复来源页码和位置；
- 浏览器返回与页面返回行为一致；
- 390、768、1024、1440px 均可完成核心路径。
