# AquaGuide 交互重构规范

本文继承工作区 `interaction-rules.md` 的全部 P0/P1 规则。

## 分层表面

| 类型 | 使用场景 | 桌面 | 手机 |
|---|---|---|---|
| 原位展开 | 规则依据、次要说明 | 卡片内展开 | 卡片内展开 |
| 详情面板 | 物种、养护、生命纪念 | 页面中央弹窗 | 底部全高面板 |
| 任务流程 | 巡检、添加、设置、AI 建缸 | 沉浸任务面板 | 沉浸全屏流程 |
| 确认弹窗 | 删除、清空、死亡登记 | 居中小弹窗 | 居中小弹窗 |

## CTA 契约

- 每个点击入口登记为 `route / view / mutation / dialog / section / external`。
- 同一区域最多一个主操作和一个不同结果的次操作。
- 卡片整卡点击与“查看详情”结果相同时删除按钮。
- 写操作必须有 loading、disabled、成功或失败反馈。
- 关闭详情恢复来源滚动、焦点与短暂高亮。

## 核心路径上限

- 每日检查：一次开始，一次进入补救文章。
- 添加生物：选择后最多三步加入。
- 水族册：一次进入，一次打开详情。
- AI 建缸助手：说目标、补信息、看方案。
