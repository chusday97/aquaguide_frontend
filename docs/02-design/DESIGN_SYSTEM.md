# AquaGuide 设计系统

> 本文从当前 `src/index.css` 与现有组件中提取，不引入新的视觉方向。设计关键词：自然、清晰、克制、可信。

## 1. 基础令牌

| 令牌 | 当前值 | 用途 |
|---|---|---|
| 页面底色 | `#FDFCF8` | 主背景，避免纯白刺眼 |
| 主文字 | `#1A1A1A` | 标题与正文 |
| 主绿色 | `#1B4D3E` | 主按钮、选中态、核心图标 |
| 浅绿色 | `#E8F0EE` | 选中背景、提示区 |
| 边界 | `rgba(0,0,0,.1)` | 卡片和分隔线 |

状态颜色需要同时配合图标与文字：成功为绿色，注意为琥珀色，高风险为红色，资料不足为中性灰。不得只靠色差表达结论。

## 2. 字体与层级

- 主字体沿用应用现有无衬线字体栈与 Geist Variable。
- 页面标题建议 28–32px，区块标题 20–24px，卡片标题 16–18px，正文 14–16px，辅助文字不小于 12px。
- 正文行高 1.5–1.7；中文正文每行建议不超过约 38 个汉字。
- 数值和状态优先使用中等字重，不用全大写英文制造层级。

## 3. 间距、圆角与层级

- 基础间距序列：4、8、12、16、24、32、48px。
- 卡片内边距：手机 16px；桌面 20–24px。
- 卡片圆角：16–24px；抽屉和大面板：约 28px；胶囊标签使用全圆角。
- 同一层级只使用一种主要阴影；内容分组优先靠间距和边界，不堆叠阴影。

## 4. 组件规范

### 按钮

| 类型 | 使用场景 | 限制 |
|---|---|---|
| 主按钮 | 当前表面的唯一下一步 | 每个视图区最多一个 |
| 次按钮 | 与主操作结果不同的次要路径 | 不得只是重复入口 |
| 文字按钮 | 展开、返回、非关键导航 | 必须有明确点击区域 |
| 危险按钮 | 删除、清空、确认死亡记录 | 只出现在确认表面 |

按钮高度至少 44px。加载时保留原宽度、禁用重复触发并显示动词状态，如“保存中”。图标按钮至少 44×44px，并提供文字标签或无障碍名称。

### 卡片

- 整卡可点击时，不再放置同结果的“查看详情”。
- 标题、状态、摘要和下一步按顺序排列。
- 内容不足时不使用空白占位维持虚假对齐。
- 物种图片使用统一宽高比与 `object-fit`，不得溢出容器。

### 状态标签

兼容判断使用固定四态：兼容、需注意、不建议、资料不足。每日检查使用日常、观察、紧急。标签文字必须与详情结论一致。

### 详情面板

固定标题区、唯一可滚动正文区、面板内底部操作区。底部操作区不能相对浏览器窗口全局固定，并要包含手机安全区。

## 5. 内容规范

- 表面标题不超过 12 个汉字，说明只保留一句。
- 首屏最多三条原因；完整规则默认折叠。
- 按钮使用“动词 + 对象”，如“查看详细判断”“保存鱼缸”。
- 避免技术术语。用户界面使用“AI 建缸助手”，不用 Copilot、BFF、fallback 等内部词。
- 风险文案说明“现在做什么”和“不要做什么”，不做疾病确诊。

## 6. 布局系统

- 桌面页面在内容容器内自适应：宽时多列，窄时降为两列或一列。
- 桌面与手机组件可以分别排版，但共享数据、状态和业务动作。
- 网格列使用可收缩最小宽度，避免固定列宽造成裁切。
- 吸顶栏、底部操作区和侧栏必须为正文预留实际空间。
- 真实手机使用单列与底部安全区；平板归入桌面布局。

## 7. 动效与反馈

- 状态切换 160–220ms；尊重 `prefers-reduced-motion`。
- 收藏、保存等轻操作可即时更新，并在失败时回滚。
- 解锁勋章只进行一次轻量庆祝，不遮挡下一步。
- 不用纯装饰漂浮、持续闪烁或长时间转场。

## 8. 可访问性基线

- 键盘焦点清晰可见，顺序与视觉顺序一致。
- 文本对比度达到 WCAG AA；大字号也不能用过淡灰色。
- 风险和选中状态同时提供文字、图标或形状线索。
- 触控目标至少 44px，目标之间保留足够间隔。
- 动态成功与错误消息通过可访问状态区域播报。
