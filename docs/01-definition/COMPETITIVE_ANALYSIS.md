# AquaGuide 竞品分析

> 调研日期：2026-07-14。仅使用产品官网、官方商店页等一手资料；功能会变化，后续迭代前需重新核对。

## 1. 竞品概览

| 产品 | 核心定位 | 公开能力 | AquaGuide 可借鉴点 |
| --- | --- | --- | --- |
| [AquaticLog](https://apps.apple.com/in/app/aquaticlog/id583895470) | 多鱼缸记录与社区平台 | 参数趋势、活体、设备、投喂、换水、提醒、照片时间线、多端同步、成就 | 参数趋势和跨端同步成熟；任务完成可单击记录 |
| [Fishkeeper](https://www.fishkeeper.co.uk/fishkeeper-app) | 鱼缸与池塘管理工具 | 无限鱼缸、物种库、任务通知、水质范围与图表、活动时间线、知识文章 | 新手/进阶同用；任务和水质记录结构完整 |
| [Aquarium Log](https://www.aquariumlog.app/about) | 简洁的水族日志 | 日历、提醒、参数监测、每日更新物种库、云备份、多鱼缸、日志分享 | 简洁记录、云备份和分享路径清晰 |
| [AqAdvisor](https://www.aqadvisor.com/) | 淡水鱼放养量计算 | 尺寸、过滤、生物数量、放养率、参数范围、换水建议和风险提示 | 单一决策任务聚焦；明确声明计算器不是长期相处保证 |

## 3. 市场机会

### AquaGuide 的差异化

1. 把物种知识、组合判断、当前鱼缸环境和实际添加动作放在同一规则链中。
2. 用每日观察把“记录工具”升级为“异常发现 → 立即动作 → 百科补救”的任务流。
3. AI 不能自行降低风险、编造文章或越过候选池，适合对安全边界敏感的新手场景。
4. 中文、小白友好的决策表达，以及 3D 鱼缸带来的直观反馈。

### 当前明显差距

1. AquaticLog、Fishkeeper 和 Aquarium Log 已把长期参数趋势、任务提醒和云同步做成基础能力；AquaGuide 当前仍以浏览器本地数据为主。
2. 当前记录偏向“有没有做”，尚缺参数趋势、照片时间线和可恢复的跨设备历史。
3. 物种详情和窄桌面布局仍有动作闭环与排版问题，会削弱规则引擎的价值。

## 4. 产品策略

- 近期：优先修复窄桌面、物种详情、成就解释和写入服务一致性，不扩张新模块。
- 中期：补齐参数历史、任务回顾、导入导出和数据恢复体验。
- 后期：评估云同步；保留“本地规则先判、AI 后解释”的差异化，不转为开放式聊天产品。
