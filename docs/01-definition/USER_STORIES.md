# AquaGuide 用户故事

## Epic 1：创建和维护鱼缸

### 1.1 创建鱼缸（P0）

作为第一次养鱼的用户，我希望按尺寸、水体、造景和设备逐步建立鱼缸，以便后续建议基于真实环境。

- [ ] 缺少必需信息时明确指出具体字段。
- [ ] 保存成功后立即成为当前鱼缸。
- [ ] 桌面和手机保存相同数据。

### 1.2 添加生物（P0）

作为缸主，我希望加入前先看到兼容性复核，以免把明显不适合的生物加入鱼缸。

- [ ] `compatible` 可直接加入。
- [ ] `caution` 必须确认后加入。
- [ ] `not_recommended` 阻断写入。
- [ ] `insufficient_data` 进入对应设置，不得绕过。

## Epic 2：查找和理解物种

### 2.1 搜索图鉴（P0）

作为新手，我希望按名称、用途和饲养条件查找物种，以便快速缩小选择。

- [ ] 搜索、筛选和分页结果一致。
- [ ] 无结果时解释原因并可清除条件。
- [ ] 卡片只有一个明确详情入口。

### 2.2 查看单个物种详情（P0）

作为购买前用户，我希望首屏知道物种身份、是否适合当前鱼缸、主要原因和下一步。

- [ ] 桌面为中央弹窗，手机为底部面板。
- [ ] 无当前鱼缸时显示一个选择/创建鱼缸引导，而非六项失败。
- [ ] “检查混养”和鱼缸设置动作产生真实跳转。
- [ ] 关闭后恢复来源卡片与滚动位置。

## Epic 3：混养决策

### 3.1 Mini 判断（P0）

作为浏览图鉴的用户，我希望选中两个物种后立即看到组合级结论，以便决定是否深入。

- [ ] 只比较所选物种，不读取当前鱼缸。
- [ ] 只显示四态结论、一条主因和唯一详细入口。
- [ ] 进入完整计算后选择不丢失。

### 3.2 完整判断（P0）

作为缸主，我希望组合结论同时考虑尺寸、设备、参数、数量和现有生物。

- [ ] 使用统一 `tank` 规则范围。
- [ ] 结论与详情、推荐和添加入口一致。
- [ ] 不建议和信息不足不能直接写入。

## Epic 4：每日观察与补救

### 4.1 每日检查（P0）

作为缸主，我希望用一分钟检查浮头、泡沫、异味、水色和行为，以便尽早发现风险。

- [ ] 首页一次点击开始。
- [ ] 本地规则先给风险、立即动作和禁止动作。
- [ ] 同缸同日重复保存更新原记录。
- [ ] 异常结果一次点击进入补救文章。

### 4.2 AI 补充解释（P1）

作为描述不够结构化的用户，我希望 AI 帮我整理描述和补充问题，但不改变系统安全结论。

- [ ] 仅异常或有自由描述时调用。
- [ ] AI 不得降低本地风险。
- [ ] AI 只能推荐本地候选文章。
- [ ] AI 失败时本地步骤仍可用。

## Epic 5：养护百科

作为遇到具体问题的用户，我希望按问题直接进入可执行步骤，而不是阅读大量无关知识。

- [ ] 搜索或分类后直接显示结果。
- [ ] 文章首屏给结论、关键步骤和检查清单。
- [ ] 收藏状态跨页面同步。
- [ ] 详情关闭后返回原结果位置。

## Epic 6：我的水族册

### 6.1 收藏与纪念（P1）

作为用户，我希望在一页查看种草、养护收藏和生命纪念，以便集中回顾。

- [ ] 四页签状态写入 URL。
- [ ] 收藏删除有确认和即时数量更新。
- [ ] 生命纪念不把死亡数量作为奖励。

### 6.2 成就勋章（P1）

作为持续养护的用户，我希望看到基于真实行为自动解锁的进度，以便知道下一步做什么。

- [ ] 明确“自动解锁，无需领取”。
- [ ] 分清已解锁、进行中和未开始。
- [ ] 每张卡只显示一个下一步。
- [ ] 旧数据能够追溯解锁。

## Epic 7：AI 建缸助手

作为不懂配置的新手，我希望描述目标并回答最多三个问题，以获得受本地规则约束的建缸方案。

- [ ] 流程为“说目标 → 补信息 → 看方案”。
- [ ] 候选只能来自本地安全或可调整池。
- [ ] 下一步动作必须真实可执行。
- [ ] AI 不可用时显示本地模板和失败原因。
