# AquaGuide QA 与验收

## 1. 验收原则

- 当前代码和真实浏览器行为优先于旧报告。
- 每个核心路径同时验证桌面、窄桌面和真实手机。
- 结论一致性优先于单页视觉完成度。
- AI 失败不能阻断确定性规则和安全下一步。

## 2. 设备矩阵

| 场景 | 预期 |
|---|---|
| 桌面 1440px | 桌面侧栏、多列内容，无溢出 |
| 桌面浏览器缩至 600px | 仍为桌面导航，内容按容器降列，无手机底栏、裁切或重叠 |
| iPad / 桌面触控设备 | 桌面布局，触控目标至少 44px |
| iPhone 环境 | 手机底栏、单列内容、底部安全区正确 |

## 3. 核心路径

### 创建与添加生物

- 桌面鱼缸顶部始终能切换鱼缸和新建鱼缸；桌面与手机均只保留鱼缸画面下方的一个正式“缸内物种”入口。
- “缸内物种”可真实展开同一内容区；添加成功后的程序化打开同样会定位、高亮并恢复焦点；3D 全屏预览可看到物种镜头列表。
- 创建鱼缸后返回概览并显示保存成功。
- 未填必填项时显示字段级错误并聚焦第一个错误。
- 添加生物保留选择，经过完整混养复核后才写入。
- 混养复核进入独立第二屏，不再显示搜索、推荐和数量长列表；返回调整后保留选择。
- 重复点击不能重复加入。

### 图鉴与混养

- 普通物种卡左上角快捷收藏不打开详情、不改变混养选择，刷新后保持并与水族册、计数和成就同步。
- 品类集合不支持一键收藏整组；进入变种列表后只能收藏具体变种。
- 选择一个物种出现 Mini 栏，两个及以上立即计算。
- 兼容、谨慎、不建议、资料不足四态均有覆盖。
- 进入完整计算后选择不丢失，并加入当前鱼缸环境。
- 详情、推荐、Mini、完整计算与添加结果使用同一规则结论。

### 每日检查

- 六个必答观察项与可选描述在同一页显示，不出现逐题下一步；选择后自动聚焦下一道未回答问题，最后一题只聚焦提交按钮。
- 覆盖正常、浮头、持续泡沫、异味、多项异常和信息不足。
- 同缸同日重复检查更新一条记录，次日创建新记录。
- 结果主操作打开白名单内养护文章，关闭后返回检查结果。
- AI 超时、未配置、非法 JSON、降级风险、虚构文章均使用安全兜底。

### 水族册与成就

- `/wishlist` 与 `/care-favorites` 正确跳到相应独立模块地址。
- `/collection` 只显示四张模块卡；四个模块地址刷新、前进后退后保持当前模块。
- 收藏、取消收藏和纪念保存后，数量与成就即时更新。
- 8 枚勋章覆盖锁定、进度、历史追溯与解锁。
- 成就说明明确“自动解锁、无需领取、还差多少”。
- 手机种草、图鉴种草和养护收藏入口均进入水族册对应页签，不打开旧弹窗或折叠列表。

### 养护文章与养护计划

- 操作型详情首屏显示可放大的主图、准备项、三步操作和禁止动作。
- 诊断型文章的 2–4 个相关问题同页显示，提交后直接进入自查结果，不保留逐题或摘要确认层。
- 手机推荐等待 4.5 秒不自动换卡，不露出左右相邻内容。
- 养护计划覆盖创建、同文章同缸更新、到期、逾期、完成、改期、确认删除和旧提醒迁移。
- 养护计划只内嵌在鱼缸页“今日行动”卡片，不出现第二张计划卡；不申请系统通知权限，也不出现独立提醒中心。
- 手机养护页与图鉴页只由应用壳提供一次底部安全空间。

### 详情与按钮

- 每个按钮产生路由、视图、数据、弹窗、区块或外部跳转结果。
- 不存在空处理、仅日志、原生 `alert` 或重复 CTA。
- 物种详情的环境补全操作进入对应设置字段。
- 从鱼缸打开已有物种时不显示重复的“查看当前鱼缸”。
- 关闭详情恢复原滚动位置、卡片与焦点。

## 4. 视觉与交互基线

- 首屏只有一个主操作和至多一个不同结果的次操作。
- 列表初次加载有结构匹配骨架；空结果有解释和有效下一步。
- 写操作有 loading、禁用、防重复、成功与失败反馈。
- 删除、清空、记录死亡有对象明确的二次确认。
- 桌面中央详情、手机底部面板和确认弹窗具备焦点限制、Esc 和返回焦点。

## 6. 发布阻断条件

- 同一组合在不同入口出现矛盾混养结论。
- AI 降低本地风险或输出白名单外补救文章。
- 桌面窄窗出现遮挡核心操作的裁切。
- 删除或死亡记录无确认直接写入。
- 保存失败无用户可见反馈。
- 正式导航出现内部 `/3d-demo` 或没有实现的概念入口。
