# 3D、物种数据与图片性能基线

> 快照日期：2026-07-14。先记录事实和退出条件，不在体验收口阶段重写用户并行修改的 3D 文件。

## 当前基线

| 项目 | 当前事实 | 判断 |
|---|---|---|
| `ThreeAquarium.tsx` 源码 | 约 44KB | 场景、纹理、交互仍集中，后续可按边界拆分 |
| `fishData.ts` 源码 | 约 831KB | 物种静态数据是主要数据体积来源 |
| 生产 3D 运行时块 | 约 889KB，gzip 约 240KB | 已延迟加载，但仍超过 500KB 构建提示 |
| 3D 页面业务块 | 约 50KB，gzip 约 16KB | 业务代码不是主要体积，运行时依赖更大 |
| 渲染策略 | `frameloop="demand"`，桌面/手机帧率封顶 | 已具备主动限帧基础 |
| 后台与离屏 | `visibilitychange` + `IntersectionObserver` | 页面后台或离屏时停止主动渲染 |
| 资源释放 | 纹理克隆后 `dispose()`；卸载时 `gl.dispose()`；动画帧取消 | 已有生命周期清理，仍需浏览器内存曲线验证 |
| 素材解析 | 2D 与 3D 都使用 `getSpeciesDisplayImage` | URL 和版本来源一致 |

## 本轮验证

- `test:3d-assets` 校验普通卡片与 3D 使用相同版本化 URL。
- 生产构建记录业务块、运行时块与物种数据相关块体积，不调高警告阈值。
- 静态检查确认后台暂停、离屏暂停、限帧、动画帧取消、纹理和 WebGL 释放路径存在。
- 批量图片另走质量审计，不与 UX 提交混合。

## 图片质量审计（2026-07-14）

- 共审计 486 张物种素材，发现 18 张中优先级返工候选，未发现高优先级问题。
- 问题分布：浅色主体低对比 11 张、主体贴近边缘 5 张、主体碎片化 2 张。
- 2026-07-15 已逐项人工复核：11 张浅色主体和 5 张边距候选的主体均完整，继续使用非破坏性表面、阴影、对比度和容器内边距保护；`sp_0357` 与 `sp_0452` 确认需要重新生成。完整记录见 `output/image_quality/manual_rework_review_2026-07-15.md`。
- 2D 卡片与 3D 贴图的版本化 URL 一致性专项测试通过。

## 下一阶段性能矩阵

| 场景 | 指标 | 通过线 |
|---|---|---|
| 首次打开有生物鱼缸 | 可交互时间、3D 首帧 | 页面操作不被 3D 阻塞；首帧失败有静态占位 |
| 低端手机 | 5 分钟平均帧率、长帧 | 目标 20–24 FPS；操作期间无持续卡死 |
| 桌面 | 5 分钟平均帧率 | 目标 28–30 FPS，符合当前上限 |
| 切到后台 | 帧调用与 CPU | 2 秒内停止主动绘制 |
| 滚出视口 | 帧调用与 CPU | 2 秒内停止主动绘制 |
| 更新素材 URL | 新旧纹理 | 加载新版，旧纹理可释放且不继续显示缓存 |
| 反复进入/离开 10 次 | JS 堆、GPU 资源 | 回落到稳定区间，不持续单调增长 |

## 拆分顺序

1. 先增加可测的首帧、暂停和释放观测点，不改变视觉。
2. 再把场景环境、纹理管理、生物交互从 `ThreeAquarium` 分离。
3. 物种数据按图鉴列表、推荐摘要、完整详情三种使用边界延后加载。
4. 若运行时仍是主要体积，评估 3D 预加载时机与静态预览，不以隐藏警告代替治理。

## 当前未完成
