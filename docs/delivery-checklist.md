# AquaGuide 交付检查清单

每次完成页面、弹窗、组件或交互改动后，必须对照本清单逐条检查，并在交付总结里用“是 / 否 / 不适用”回答。凡是回答“否”的项目，需要列出具体问题和下一步修改项。

## 必查项目

| 检查项 | 回答要求 | 需要重点检查的场景 |
| --- | --- | --- |
| 异步操作有 loading 态 | 是 / 否 / 不适用 | 保存、生成、上传、AI 请求、数据库请求、图片导出、搜索加载 |
| 操作结果有反馈提示 | 是 / 否 / 不适用 | 添加成功、保存成功、复制成功、删除成功、失败提示、toast、弹窗确认 |
| 表单有错误处理 | 是 / 否 / 不适用 | 必填为空、数量非法、日期非法、网络失败、字段格式错误 |
| 破坏性操作有二次确认 | 是 / 否 / 不适用 | 删除鱼缸、移除生物、清空数据、导入覆盖、重置配置、记录死亡 |
| 空状态有提示 | 是 / 否 / 不适用 | 无搜索结果、无收藏、空鱼缸、无换水记录、无推荐、无诊断结果 |

## 交付总结格式

```text
交付检查：
- 异步操作有 loading 态：是 / 否 / 不适用
- 操作结果有反馈提示：是 / 否 / 不适用
- 表单有错误处理：是 / 否 / 不适用
- 破坏性操作有二次确认：是 / 否 / 不适用
- 空状态有提示：是 / 否 / 不适用

需要修改的问题：
- 无 / 或列出具体问题
```

## 判断规则

- 如果本次没有新增或修改异步操作，可以回答“不适用”。
- 如果本次没有新增或修改表单，可以回答“不适用”。
- 如果本次没有新增或修改删除、清空、覆盖等破坏性操作，可以回答“不适用”。
- 如果页面或组件可能出现“没有数据”的情况，但没有对应提示，应回答“否”。
- 如果点击后用户无法明确知道成功、失败或正在处理中，应回答“否”。
